7. command: tail -f system.log 2>/dev/null | grep --line-buffered "ERROR" >>error_report.txt
   observation: 1.tail -f system.log 
                  tail it displays end of the file and -f monitors system.log continuously, added a new line in system.log in the real time.
               2. 2>/dev/null 
                  2 represents standard error stream, This > operator redirect the output and /dev/null is  a file that stops anything written
                  to it. supress the error message means system.log not opened in terminal.
               3. |
                  This is a pipe operator  it sends the ouptput of Left side tail -f directly to right side grep without creating a temporary 
                  file.
               4. grep --line-buffered "ERROR"
                   grep search that line  which conatining the word "ERROR". The --line-buffered option forces grep to output matching lines 
                   immediately rather than waiting for the buffer to fill. This is ideal for real-time log monitoring.
               5. >> error_report.txt
                  >> error_report.txt redirects the output containing the matching lines with the word "ERROR", and appends them to the file 
                     error_report.txt.
